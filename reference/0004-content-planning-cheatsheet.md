# 内容规划能力速查表

## 内容类型金字塔

```
        🏔️ 版本内容
       (每季/每月大事件)
      ─────────────────
     🌊 周期内容
    (周常/竞技赛季/活动)
   ─────────────────
  💧 日常内容
 (每日任务/日常副本)
─────────────────
 🪨 基础内容
(核心玩法/核心循环)
```

## 内容消耗速度公式

```
内容消耗速度 = 玩家日均游戏时长 × 效率
内容生产速度 = 团队产能 × 开发周期

健康条件：内容生产速度 ≥ 内容消耗速度
```

## 版本节奏框架

```
大版本 (3-4 月) → 新地图/新系统/新等级
          ↓
小版本 (2-4 周) → 新活动/新英雄/新副本
          ↓
迷你活动 (1-7 天) → 节日活动/限时挑战
          ↓
日常内容 (每天) → 每日任务/签到
```

## 活动日历设计模板

| 时间 | 周 | 活动 | 目标 |
|------|----|------|------|
| 版本第 1 周 | 1 | 新剧情/新副本 | 核心玩家体验新内容 |
| 版本第 2 周 | 2 | 高难挑战开放 | 硬核玩家 |
| 版本第 3 周 | 3 | 社交活动(公会战) | 社交玩家 |
| 版本第 4 周 | 4 | 双倍掉落/Buff 周 | 休闲玩家追赶 |
| 版本第 5 周 | 5 | 收尾活动 → 预告下版本 | 维持活跃到版本末 |

## 内容规划检查清单

- [ ] 核心循环能否支撑玩家每天 15-30 分钟的日常？
- [ ] 是否有周常/周期内容给玩家每周目标？
- [ ] 大版本间隔是否超过玩家内容消耗速度？
- [ ] 内容空窗期是否有填充活动？
- [ ] 不同玩家类型（休闲/核心/硬核）是否都有内容？
- [ ] 新内容是否会淘汰旧内容？
- [ ] 是否有"追赶机制"帮助新玩家赶上进度？

## 常见内容设计模式

| 模式 | 描述 | 游戏例 |
|------|------|--------|
| **日常收菜** | 每天 10-30 分钟做完日常 | 原神每日、FGO 登录 |
| **周常收菜** | 每周一次的高奖励内容 | 周本、公会战 |
| **赛季制** | 3-4 月重置进度 | 炉石天梯、COC 传奇杯 |
| **限时活动** | 2 周的限定内容 | 节日活动、联动 |
| **肉鸽/随机** | 每次不同的体验 | Hades、以撒 |
| **玩家生成** | 玩家创造内容 | Roblox、马造 2 |

## 避免的陷阱

1. **内容断档** — 玩家在等的期间大量流失
2. **内容过剩** — 玩家跟不上版本，产生焦虑
3. **旧内容完全作废** — 玩家的时间投入被归零（挫败感）
4. **缺乏追赶机制** — 新玩家永远追不上老玩家
5. **长草期过长** — 无事可做 → 流失
