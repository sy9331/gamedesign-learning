# 游戏经济设计速查表

## 经济三要素

```
产出 (Source)  ──→  流通 (Trade)  ──→  消耗 (Sink)
     ↑                                  │
     └──────────────────────────────────┘
```

- **Source**：资源从哪里来（副本掉落、任务奖励、每日产出）
- **Sink**：资源从哪里消失（强化、合成、修理、手续费）
- **流通**：资源如何在玩家之间流动（交易行、赠送、邮寄）

## 核心法则

> **Source − Sink > 0 → 通货膨胀**
> **Source − Sink < 0 → 通货紧缩**
> **Source − Sink ≈ 0 → 健康经济**

## 资源类型金字塔

```
Layer 1: 硬通货 (Hard Currency)
  - 充值货币（钻石、点券）
  - 有限产出，链接现实货币
  - 来源唯一：付费

Layer 2: 软通货 (Soft Currency)
  - 游戏内主要流通货币（金币）
  - 大量产出，大量消耗
  - 来源：日常、副本、活动

Layer 3: 代币 (Token)
  - 专项货币（公会币、竞技场币、荣誉）
  - 锁定在特定系统内流通
  - 目的：隔离经济，防止通兑

Layer 4: 材料 (Materials)
  - 特定用途的资源（强化石、碎片）
  - 不可交易 / 有限交易
  - 目的：控制成长节奏
```

## 经济健康指标

| 指标 | 含义 | 预警值 |
|------|------|--------|
| **Gini 系数** | 玩家之间的财富差距 | >0.5 = 贫富分化严重 |
| **货币流通速度** | 单位货币的周转次数 | 过低=通缩，过高=通胀 |
| **Source/Sink 比** | 产出/消耗比例 | 长期>1.2 = 通胀 |
| **Stack 率** | 玩家手中囤积的资源量 | 持续增长 = 缺少 Sink |

## 经典经济设计模式

### 模式 1：Sink 绑定成长（COC 模式）
```
打金 → 升级建筑（消耗金币） → 打更多金
资源仓库有上限 → 不打就浪费 → 被迫上线的驱动力
```

### 模式 2：资源分层隔离（原神模式）
```
原石（付费）→ 抽卡（专用消耗）
摩拉（通用）→ 强化、锻造
树脂（体力）→ 副本准入
经验书（专用）→ 角色升级
```
每层不互通 → 控制通兑风险

### 模式 3：社交驱动流通（MMO 模式）
```
玩家 A 采药 → 挂拍卖行 → 玩家 B 购买 → 制作药水 → 卖回给 A
经济是社交的"骨架"
```

## Sink 设计工具箱

| Sink 类型 | 例子 | 效果 |
|-----------|------|------|
| **成长消耗** | 强化、升级、突破 | 稳定消耗，可控节奏 |
| **手续费** | 交易税、邮寄费 | 持续小额消耗 |
| **耐久度** | 修理装备 | 强迫循环 |
| **养成消耗** | 技能升级、天赋 | 大额一次性消耗 |
| **社交消耗** | 公会捐献、送礼 | 社交绑定 |
| **装饰消耗** | 时装、外观、坐骑 | 炫耀型消耗 |
| **重置消耗** | 洗点、重置天赋 | 错误成本 |

## 通胀预警信号

- [ ] 玩家抱怨"钱不值钱"
- [ ] 交易所价格持续上涨
- [ ] 新玩家追赶成本越来越高
- [ ] 老玩家囤积了大量货币无事可做
- [ ] 工作室大量涌入（印钱速度超过设计预期）

## 一句话总结

> **游戏经济设计 = 控制资源从哪里来（Source）和到哪里去（Sink），让玩家的"努力"和"收获"保持被感知的价值。**
