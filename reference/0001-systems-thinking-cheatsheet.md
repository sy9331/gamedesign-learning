# 系统思维速查表

## 核心概念

```
游戏系统 = 一组相互连接的元素 → 产生单独元素不具备的行为
```

## 核心循环 (Core Loop)

```
玩家行动 → 获得奖励 → 成长变强 → 迎接新挑战
                                         ↑
                                         └── 回到行动，但难度提升 ──┘
```

**检查清单**：
- [ ] 这个循环的核心动作是否有趣？
- [ ] 循环的闭环周期是否合适？（太短→无聊，太长→疲劳）
- [ ] 每个环节是否有明确的正向反馈？

## 反馈回路

| 类型 | 效果 | 游戏例子 |
|------|------|---------|
| **正反馈 🔥** | 放大差距，滚雪球 | MOBA 击杀→经济→装备→更易击杀 |
| **负反馈 🧊** | 缩小差距，追赶机制 | 马里奥赛车蓝龟壳、LOL 赏金、抽卡保底 |

**设计原则**：正反馈制造爽感，负反馈保持悬念。两者平衡 = 好设计。

## 涌现 (Emergence)

```
简单规则 × 组合 × 玩家创造力 = 无限可能性（免费内容）
```

**涌现的三个条件**：
1. 规则足够简单（玩家能理解）
2. 组合空间足够大（玩家能探索）
3. 有关联性反馈（玩家能发现新玩法）

## 分析框架（拆解任何游戏）

### 第一步：找到核心循环
Q: "玩家在这游戏里到底在重复做什么？"

### 第二步：识别反馈回路
Q: "什么让领先者更领先？什么让落后方有机会？"

### 第三步：寻找二阶效应
Q: "如果我改动 X，除了直接效果之外，还会影响什么？"

### 第四步：检查涌现空间
Q: "玩家能做出我没设计过的玩法吗？"

## 常见陷阱

🕳️ 只看一阶效果，忽略连锁反应
🕳️ 把"功能"当"系统"——每个功能都是系统的一部分
🕳️ 只加正反馈不加负反馈（滚雪球到死）
🕳️ 把所有路径写死（扼杀涌现）

## 一句话总结

> **系统思维是让你在推倒第一块多米诺骨牌之前，看清整条链的能力。**
