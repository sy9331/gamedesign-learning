🧠 系统思维：游戏策划的"元能力"

如果只学一种能力，那就是系统思维。它是所有其他模块的基础。

学习进度 8 个模块 · 当前: 1/8
系统思维 ◀ 玩家心理学 经济设计 内容规划 决策能力 商业化 长线运营 产品分析

🎯 为什么系统思维是所有能力的基石？

想象这个场景：

你是某卡牌游戏的策划。运营同学说："最近付费率有点低，我们把月卡从 30 块降到 18 块吧。"

没有系统思维的策划会说："好主意！降价肯定卖更多。"

有系统思维的策划会问：

  • 降价后老玩家会不会觉得被背刺（公平感）？
  • 月卡用户多了，免费玩家的 PvP 体验会不会崩？
  • 收入目标是涨还是维持？降价的缺口靠什么补？
  • 月卡降价了，季卡/周卡要不要调？
  • 这次降价对三个月后的周年庆活动有什么影响？
核心洞察 游戏里没有一个改动是孤立的。每一个决策都会像多米诺骨牌一样推倒其他系统。系统思维就是让你在推第一块牌之前，看清整条链的能力。

"A game is a system in which players engage in an artificial conflict, defined by rules, that results in a quantifiable outcome."

— Salen & Zimmerman, Rules of Play

1️⃣ 什么是游戏系统？

📖 定义

系统 = 一组相互连接的元素，它们共同运作产生出单独元素不具备的行为。

游戏里的"一个功能"从来不是一个功能——它是一个系统。比如《暗黑破坏神》的"暴击"：

装备暴击率 → 暴击触发 → 暴击伤害 → 怪物死亡 → 掉落装备 ↕ 装备暴击率↑

这不是一条直线——是一个回路。理解回路，就是系统思维的第一步。

2️⃣ 核心循环 (Core Loop)

🎮 定义

核心循环是玩家在游戏中最基本的重复行为循环。它定义了游戏的本质体验——"玩家到底在做什么"。

每一个游戏都有一个最底层的循环：

游戏 核心循环
Pokémon GO 走路 → 抓宝可梦 → 进化/强化 → 打道馆 → 需要更强 → 走路
原神 消耗树脂 → 打 Boss → 获得素材 → 强化角色 → 打更强 Boss → 等树脂恢复
COC 部落冲突 造兵 → 攻击 → 获得资源 → 升级建筑 → 造更强兵 → 攻击
大乱斗 选择角色 → 对战 → 看到结果 → 想试试别的角色 → 选择角色
三消 (Candy Crush) 消除 → 得分 → 通关 → 新关卡 → 消除
🎯 行动
(打怪) → 📦 奖励
(经验/装备) → 📈 成长
(变强) → 🎯 新挑战
(更强敌人)
▲ 经典 RPG 核心循环 —— 回到起点，但难度提升 ▲
⚡ 黄金法则：如果一个游戏的核心循环出了问题，其他所有系统都救不了它。

面试高频

面试中的经典问题："请分析某游戏的系统结构。"——几乎所有好答案都从核心循环开始。

3️⃣ 反馈回路 (Feedback Loops)

🔄 定义

系统中一个变量的变化，通过一系列链条最终影响自身。这是系统思维里最重要的概念，没有之一。

🔥 正反馈 (Positive Feedback) — 放大差异

"强者愈强"——成功带来更多成功。

🏆 击杀成功 → 💰 获得经济 → 🛡️ 更好装备 → 🏆 更容易击杀
游戏场景正反馈链条
MOBA 经济领先击杀 → 金钱 → 装备 → 更容易击杀 → 滚雪球
MMO 等级压制等级高 → 能打高级怪 → 经验更多 → 等级更高
三消连消消除 → 得分 → 触发额外消除 → 更多分数
SLG 兵力滚雪球打赢 → 抢资源 → 造更多兵 → 打更多人

正反馈是好是坏？ 看设计目的：

✅ 适合用正反馈

  • 想让游戏快速结束（大乱斗的怒火条）
  • 让玩家感受到"我变强了"（爽感）
  • 鼓励竞争排名（排行榜的头部效应）

⚠️ 正反馈过强的问题

  • 领先方滚雪球 → 弱方没体验
  • 新手永远追不上老玩家
  • 比赛失去悬念（垃圾时间）

🧊 负反馈 (Negative Feedback) — 缩小差距

"追赶机制"——让落后方有机会翻盘，保持悬念。

游戏场景负反馈机制
马里奥赛车第一名吃不到好道具，最后一名拿到闪电/蓝龟壳
LOL 赏金机制连续击杀的玩家被终结时，给对面大量金币
抽卡保底抽得越多，隐性概率越高（保底机制）
COC 匹配机制匹配与你实力相近的对手，而不是最强的
MMO 经验衰减等级超过怪物时经验减少 → 迫使你去新区域

⚖️ 高手策划同时使用两者

🔥 正反馈 你变强了（爽感）
🧊 负反馈 保持悬念（公平感）
⚖️ 两者平衡 = 好的竞技游戏

💡 实战技巧：当你的游戏出现"一边倒"时，检查是不是正反馈过强而缺少负反馈。当你感觉"努力了也白费"时，检查是不是负反馈过强而正反馈不足。

4️⃣ 涌现 (Emergence)

✨ 定义

涌现 = 简单的规则组合在一起，产生了复杂且不可完全预测的行为。这是系统思维最美妙的部分——也是游戏设计者的"免费内容"。

经典例子 — 剪刀石头布：3 条规则 → 无限的策略（读心、概率、模式识别）。

游戏中的涌现案例：

  • 《塞尔达：旷野之息》：物理引擎 + 元素系统 + 道具 → 玩家用"金属武器引雷劈敌人"——没有设计师专门设计过这个玩法
  • 《Dota 2》：100+ 英雄 × 各自技能 → 每局都不同的团战组合 —— "刷新球 + 跳刀"的连招从未被单一设计师预见到
  • 《Minecraft》：红石 + 方块 + 活塞 → 玩家造出了可运行的计算机
  • 《Hades》：不同神的祝福组合 → "猎人流""雷人流"——玩家发现的配装远超设计师的预期
为什么涌现是"免费内容" 你不需要设计每一种可能性。只需要设计会产生可能性的规则。玩家自己会创造内容，而且他们创造的内容永远比你设计的更多。
⚠️ 反面案例：有些手游把所有玩法路径都写死了（自动寻路、自动战斗、一键完成）→ 没有任何涌现空间 → 玩家迅速厌倦 → 你需要不断出"新系统"来弥补 → 游戏越来越臃肿。这就是缺乏涌现的代价。

5️⃣ 意外后果 — 真实案例分析

系统思维的真正价值：预判别人看不到的连锁反应。

📋 案例 1：《Diablo 3》拍卖行 (2012)

设计意图：让玩家安全交易装备，官方抽成赚收入。

系统后果：

  1. 玩家发现拍卖行买装备比自己打怪效率高 10 倍
  2. 没人打怪 → "打怪爆装备"的核心循环断裂
  3. 大量玩家直接跳过游戏内容，只盯着拍卖行价格
  4. 暴雪最终关停拍卖行（承认这是游戏历史上最大的错误之一）

教训：经济系统会影响核心循环。他们只看到了"收入"，没看到"对打怪动机的冲击"。

📋 案例 2：某国产手游"首充双倍重置"

设计意图：每次版本更新重置双倍 → 刺激付费。

系统后果：

  1. 玩家学会"只在重置当天充值" → 日常收入断崖式下跌
  2. 非重置日的活动充值意愿极低
  3. 整个游戏的收入节奏被促销周期绑架

教训：促销节奏如果没有和内容消耗节奏配合，会扭曲玩家的付费习惯。你训练了玩家只在特定时间花钱。

📋 案例 3：《Rise of Kingdoms》王国 vs 移民

设计意图：允许玩家移民到新王国 → 增加社交活力。

系统后果：

  1. 最强玩家集中移民到一两个王国
  2. 老王国变成死服（活跃玩家流失）
  3. 新玩家进入死服 → 体验差 → 流失
  4. 氪金玩家集中 → 小王国无法竞争 → 跨服战失衡

教训：玩家会用最优解玩你的系统。如果"最优解"是离开你的老服务器，那你的社交系统设计就有根本问题。

三条教训的共同点 三个案例的策划都只看到了系统的直接效果（卖装备赚钱、促销拉升收入、移民拉动活跃），没看到系统的二阶效应（对核心循环的破坏、对行为的扭曲、对生态的稀释）。系统思维就是把你从"一阶思考"提升到"二阶、三阶思考"。

🛠️ 实践练习

✏️ 练习：拆解你的游戏

选一款你最熟悉的游戏，完成以下三个任务：

步骤 1：画出核心循环

用箭头连接：行动 → 奖励 → 成长 → 新挑战（或类似结构）

步骤 2：识别至少一个正反馈回路

描述"什么让领先者更领先"？例如：等级高 → 打怪快 → 升级更快 → 等级更高

步骤 3：识别至少一个负反馈回路

描述"什么在追赶差距"？例如：PvP 匹配机制把同分段的人分在一起

💡 试着用游戏里具体的数值或机制来写，而不是笼统的描述。例如：

《LOL》的正反馈：击杀获得 300-500 金币 → 购买装备 → 伤害提升约 15-20% → 更容易击杀。负反馈：连续击杀的玩家有赏金（250-1000 金），被终结时给对面巨额经济。

《游戏设计艺术》 (The Art of Game Design) — Jesse Schell

有中译本（第 2 版），豆瓣评分 9.2

《系统之美》 (Thinking in Systems) — Donella Meadows
不是游戏书，但它是系统思维最好的入门。非常薄，3-4 小时能读完。
中文版，200 页左右，适合通勤读
3
《游戏设计梦工厂》 (Game Design Workshop) — Tracy Fullerton
比 Schell 更实操，第 3 部分专门讲系统分析。
有中译本，适合配合练习使用
